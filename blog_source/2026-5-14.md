本文章描述了如何在 Linux 上编译 LineageOS 23.2 并安装到虚拟机。

这篇文章分为几个步骤：下载源代码、编译、安装。

# 准备

## 环境

我的编译环境：Arch Linux (linux kernel 7.0.5)、24核CPU+32G内存+1T固态硬盘。

按照官方文档的说法，在 Ubuntu 上编译需要下载以下软件包：

`bc bison build-essential ccache curl flex g++-multilib gcc-multilib git git-lfs gnupg gperf imagemagick protobuf-compiler python3-protobuf lib32readline-dev lib32z1-dev libdw-dev libelf-dev libgnutls28-dev lz4 libsdl1.2-dev libssl-dev libxml2 libxml2-utils lzop pngcrush rsync schedtool squashfs-tools xsltproc zip zlib1g-dev`

不过我这里是 Arch Linux，日常使用的时候可能都已经齐了，所以我就不管了。

如果编译 lineageos-22.0 以上的版本，还需要安装下面的软件包：

`meson glslang-tools python3-mako`

## 下载 repo 工具

```bash
curl https://storage.googleapis.com/git-repo-downloads/repo > ~/.local/bin/repo
chmod a+x ~/.local/bin/repo
```

之后可以根据需要添加到 `PATH`。（不过可以不用添加，因为只用这一次）

## 配置 lfs

有一些项目是 lfs 项目，需要配置 lfs。

```bash
git lfs install
```

## 配置 ccache

ccache 用于加快 Native C/C++ 代码的构建速度，原理大概是把已经编译好的文件缓存起来，等到下次需要的时候不需要重新编译。（此步骤可选）

需要安装 `ccache` 软件包，然后进行如下配置。

```bash
ccache -M 64G # 启用 64G 缓存
ccache -o compression=true # 启用压缩（可选）
```

# 下载源代码

由于我要编译最新的 lineageos-23.2，北外源还没有更新这么快，所以我先用北外源下载大部分，剩下的再去 Github 上下载，这样更快。

另外源码占用空间很大，建议提前备好 400G 空间（因为还有编译的预留空间）。

```bash
mkdir ~/Build/LineageOS
cd ~/Build/LineageOS
```

初始化仓库

```bash
repo init -u https://github.com/LineageOS/android.git -b lineage-23.2 --git-lfs --no-clone-bundle
```

之后按 LineageOS北外源 的说法进行如下更改：

打开`.repo/manifests/default.xml`，将

```
  <remote  name="github"
           fetch=".."
           review="review.lineageos.org" />
```

改成

```
  <remote  name="github"
           fetch="https://github.com/" />

  <remote  name="lineage"
           fetch="https://mirrors.bfsu.edu.cn/git/lineageOS/"
           review="review.lineageos.org" />
```

替换 AOSP 请参考 [AOSP 帮助](https://mirrors.bfsu.edu.cn/help/AOSP/) 中「LineageOS 中对于 AOSP 的替换」一节

将

```
  <default revision="..."
           remote="github"
```

改成

```
  <default revision="..."
           remote="lineage"
```

之后运行 `repo sync` 同步。

过程可能花费数天时间，因为我自己的流量是计费的，只能用校园网，所以挺慢的。当然如果你是千兆大带宽当我没说

~~过了几天~~之后就会报错，因为我同步的是 lineageos-23.2，北外源上没有，报错了之后多试几次，确保尽可能多同步点；之后把链接改成 Github 的链接接着同步。

（改成 Github 链接的方法：上述过程的逆过程）

> 当然如果你编译的没有这么新，可以忽略这些，直接同步北外源的

# 编译

## 准备

先设置一些环境变量。

如果启用了 ccache，需要下面的设置：

```bash
export USE_CCACHE=1
export CCACHE_EXEC=/usr/bin/ccache
```

如果需要禁用 A/B 分区方案：（真机大概都是 A/B 分区的，不过在虚拟机里变砖之类的都没太大关系，直接删除重建就能解决变砖问题。禁用了之后还能节省空间）

```bash
export AB_OTA_UPDATER=false
```

构建下表中的一个目标，运行以下命令

```
breakfast <目标>
```


| 目标                 | 架构                 | 类型       |
| -------------------- | -------------------- | ---------- |
| virtio\_arm64        | ARM（32 位 + 64 位） | PC         |
| virtio\_arm64only    | ARM（仅 64 位）      | PC         |
| virtio\_x86\_64      | x86（仅 64 位）      | PC         |
| virtio\_x86\_64\_car | x86（仅 64 位）      | 汽车       |
| virtio\_x86\_64\_tv  | x86（仅 64 位）      | Android TV |

这里我选择 virtio_x86_64。

**注意：** LineageOS 23.x 编译还需要设置 `TARGET_RELEASE` 环境变量，而 `breakfast` 命令不会设置这个环境变量，这时需要使用 `lunch` 命令。构建 `virtio_x86_64` 的命令如下：

```bash
lunch lineage_virtio_x86_64 bp4a userdebug
```

其中 `userdebug` 也可以换成 `user` （用户模式）或 `eng` （开发模式）。具体可以去网上搜

## 编译

运行

```bash
m isoimage-install -j6
```

其中核心数根据自己的配置酌情选择：编译过程中会使用 clang 和 java 编译，java 是内存消耗大户，如果直接把 -j 设满的话每个 Java 进程都会占 4G 内存，内存就爆了，加 swap 也会降低速度，所以不建议加过多的 swap。

结果会出现在 `out/target/product/<target>/lineage-*-20260215-UNOFFICIAL-<target>.iso`。</target></target>

## 坑

```bash
Error converting to codepage 850 Invalid argument
```

之前加过 `export AB_OTA_UPDATER=false`，后来忘加了的问题，不加会导致虚拟机不能启动，这点还需留意

# 安装

打开 virt-manager，新建虚拟机，选择合适的架构之后选择自己的 ISO

![](https://res.imitoy.top/Qexo/26/5/1778768808.854898.png)

之后设置内存和 CPU 数量。内存至少为 2G。

![](https://res.imitoy.top/Qexo/26/5/1778768911.726439.png)

之后取消勾选“为虚拟机启用存储”，存储一会再配置。

之后打开配置面板，添加硬件，先添加两个硬盘，第一个硬盘是系统分区，第二个是存储。需要把 SATA 改成 VirtIO。
![](https://res.imitoy.top/Qexo/26/5/1778769037.906311.png)

如果开启了 A/B 分区，系统分区至少需要 13G，否则最少需要 5G。存储分区可以自由分配，建议不低于 2G。

之后开启虚拟机，进入 Recovery

![](https://res.imitoy.top/Qexo/26/5/1778769319.133493.png)

选择恢复出厂设置

![](https://res.imitoy.top/Qexo/26/5/1778769337.596702.png)

然后回到最开始的界面（按 ESC），选择 Apply Update -> Choose from ISOIMAGE。（当然如果此时你有别的 OTA 升级包，也可以使用 adb 侧载，IP 地址屏幕上就有，adb 能直接连）

更新之后系统引导是从第一个系统分区的虚拟磁盘引导的，需要在虚拟机配置中的引导选项中选择第一个虚拟磁盘。

之后启动

![](https://res.imitoy.top/Qexo/26/5/1778769548.504605.png)

Settings 里可以进行一些设置，屏幕分辨率啥的，一般不用设置，直接进第一个就行

首次启动加载时间可能会长，中间屏幕卡住半分钟属于正常现象

通过新手引导之后就可以体验 LineageOS 了

![](https://res.imitoy.top/Qexo/26/5/1778769651.362241.png)
